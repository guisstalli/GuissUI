==================================================
CENTRAL MODULE — PATIENT MEDICAL RECORD
==================================================

Definition:
A Patient Medical Record (Dossier Patient) is a longitudinal container.
It aggregates all information related to a patient over time.
It does NOT create medical data and does NOT replace exams or consultations.

The Patient Medical Record MUST be accessible from:
- Patient list
- Consultations
- Exams
- Appointments

--------------------------------------------------
PATIENT MEDICAL RECORD – GLOBAL LAYOUT
--------------------------------------------------

Design a Patient Medical Record screen.

TOP HEADER:
- Patient identity (read-only)
  - Full name
  - Patient ID
  - Age
  - Sex
  - Child / Adult indicator

LAYOUT:
- Horizontal tabs or vertical sections

SECTIONS:
- Summary
- Medical History
- Consultations
- Exams
- Appointments & Follow-up
- Documents

--------------------------------------------------
SECTION: SUMMARY
--------------------------------------------------

Purpose:
Provide an instant clinical overview.

CONTENT (read-only):
- Last consultation date and status
- Last exam (type, date, status)
- Active diagnoses (aggregated)
- Next appointment (if any)
- Alerts (incomplete exam, follow-up required)

--------------------------------------------------
SECTION: MEDICAL HISTORY
--------------------------------------------------

- Display antecedents (ICD-based)
- Show last update date
- Edit action redirects to Medical History module

--------------------------------------------------
SECTION: CONSULTATIONS
--------------------------------------------------

- Chronological table
- Date, status, linked exam
- Access to consultation detail

--------------------------------------------------
SECTION: EXAMS
--------------------------------------------------

- Child exams and Adult exams clearly separated
- Chronological order
- Status (Completed / Incomplete)
- Access to exam view (read-only)

--------------------------------------------------
SECTION: APPOINTMENTS & FOLLOW-UP
--------------------------------------------------

- Upcoming appointments
- Past appointments
- Status (planned, cancelled, no-show)

--------------------------------------------------
SECTION: DOCUMENTS
--------------------------------------------------

- Reports (PDF)
- Medical images
- Export history

RULES:
- No editing of clinical data inside the Patient Medical Record
- All edits happen in dedicated modules

==================================================
MODULE — CONSULTATIONS
==================================================

Definition:
A consultation represents a medical visit context.
It links a patient, a date/time, and optionally an exam.
It does NOT contain clinical data itself.

Statuses:
- Planned
- In progress
- Completed
- Cancelled

--------------------------------------------------
CONSULTATIONS LIST SCREEN
--------------------------------------------------

Design a consultations list screen.

COLUMNS:
- Date
- Time
- Patient name
- Age
- Consultation type (Child / Adult)
- Status
- Linked exam
- Action (View / Continue)

FILTERS:
- Date range
- Status
- Child / Adult
- With exam / Without exam

--------------------------------------------------
CONSULTATION DETAIL SCREEN
--------------------------------------------------

TOP SECTION:
- Patient identity
- Consultation date and time
- Status
- Consultation type

SECTIONS:
- Linked appointment
- Linked exam (if exists)
- Administrative notes (non-clinical)

ACTIONS:
- Start consultation
- Create or link exam
- Cancel consultation

==================================================
MODULE — APPOINTMENTS (RENDEZ-VOUS)
==================================================

Definition:
An appointment is a scheduled time slot.
It contains no clinical data.

--------------------------------------------------
APPOINTMENT CREATION MODAL
